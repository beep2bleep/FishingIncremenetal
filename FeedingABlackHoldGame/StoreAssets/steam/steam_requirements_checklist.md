Steam Store Submission Checklist
Source: Steamworks docs (store assets + written description)

Required store graphic assets (current standard sizes):
- Header Capsule: 920 x 430
- Small Capsule: 462 x 174
- Main Capsule: 1232 x 706
- Vertical Capsule: 748 x 896

Library assets (recommended for better presentation):
- Library Capsule: 600 x 900
- Library Header: 920 x 430
- Library Hero: 3840 x 1240
- Library Logo: transparent PNG (Steam does not enforce one single fixed pixel size in docs; supply a clean high-resolution source)

Written store fields to prepare:
- Short description
- About This Game section
- Supported languages list
- Capsule and screenshot set aligned with copy promises

Formatting notes:
- Steam supports a BBCode subset in the About This Game section.
- Keep promises specific to implemented gameplay systems.
- Avoid UI text overload in capsule art.

Suggested screenshot set (6 to 10):
1) Early wave combat with full HUD
2) Multi-hero lineup in active fight
3) Active ability moment (visual spike)
4) Boss encounter with segment health progress
5) Upgrade screen / progression tree
6) Reward spike moment (coins/power/event)

Store page QA before publish:
- Short description immediately communicates genre + hook
- First 2 screenshots show real in-game action, not menus
- About section has scan-friendly bullets and no vague filler
- Visual style is consistent across all capsules
